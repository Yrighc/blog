Git 常用命令高频速查

这篇文章整理了在日常开发中非常高频使用的 Git 命令，配合目录大纲（TOC）方便快速检索。 1. 基础操作 1.1 暂存与提交 将修改添加至暂存区，并提交到本地仓库： # 添加所有修改的文件 git add . # 提交并附带说明信息 git commit -m "feat: add new features" 1.2 查看状态 随时检查当前工作区的状态： git status 2. 分支管理 2.1 创建与切换 在开发新功能时，通常需要建立独立的分支： # 创建并切换到新分支 git checkout -b feature/login # 切换回主分支 git checkout main 2.2 合并分支 将开发分支合并回主分支： git merge feature/login 3. 远程同步 3.1 推送更改 把本地提交推送到 GitHub： git push origin main 3.2 拉取更新 同步远程仓库的最新代码并与本地分支合并： git pull origin main ↩️ 相关页面 返回知识库主页：首页 了解关于我：关于我
